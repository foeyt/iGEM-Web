# 🤖 AI 辅助开发最佳实践

AI 已经成为现代开发者的得力助手。本指南将教你如何**有效、负责任地**使用 AI 提高开发效率。

## 🎯 核心原则

### ✅ 正确的 AI 使用方式

1. **AI 是助手，不是替代品**
   - 让 AI 解释概念和代码
   - 让 AI 提供多种解决方案
   - 让 AI 优化你的代码

2. **必须理解每一行代码**
   - 不要直接复制粘贴大段代码
   - 逐行阅读 AI 生成的代码
   - 能够向他人解释代码逻辑

3. **测试是必须的**
   - AI 生成的代码可能有 bug
   - 在集成前必须测试
   - 理解代码的副作用

### ❌ 错误的 AI 使用方式

- ❌ 完全依赖 AI，不思考
- ❌ 复制代码后不测试就提交
- ❌ 无法解释自己项目的代码
- ❌ 把 AI 当成"答案生成器"
- ❌ 让 AI 一次性生成整个项目

## 💡 AI 提示词（Prompt）技巧

### 基础提问格式

**❌ 不好的提问**：
```
怎么写一个网页？
React 怎么用？
这个代码有问题
```

**✅ 好的提问**：
```
我想用 HTML 和 CSS 创建一个响应式的导航栏，
包含 Logo 和 3 个导航链接，在移动端显示为汉堡菜单。
请给我示例代码并解释原理。
```

### 提问模板

#### 1. 学习新概念

```
我想学习 [技术/概念]，我目前的水平是 [基础/中级]。
请用简单的语言解释 [技术/概念] 是什么，
并给我一个简单的示例代码。
```

**示例**：
```
我想学习 React 的 useState Hook，我已经学过基础 JavaScript。
请用简单的语言解释 useState 是什么，
并给我一个计数器的示例代码。
```

#### 2. 解决 Bug

```
我在做 [项目描述]，遇到了以下错误：
[错误信息]

我的代码：
[粘贴相关代码]

我已经尝试了 [你的尝试]，但还是不行。
请帮我找出问题并解释原因。
```

#### 3. 代码优化

```
这是我写的 [功能描述] 代码：
[粘贴代码]

请帮我：
1. 优化代码性能
2. 改进可读性
3. 指出潜在问题
并解释每个改进的原因。
```

#### 4. 设计建议

```
我想设计一个 [页面/组件] 包含 [功能描述]。
目标用户是 [用户群体]。
请给我设计建议和实现思路。
```

## 🛠️ 分阶段 AI 使用指南

### 第一阶段：HTML + CSS

**适合问 AI 的问题**：

1. **学习基础概念**
```
什么是语义化 HTML？为什么要用 <header> 而不是 <div>？
请举例说明。
```

2. **布局实现**
```
我想实现一个两列布局，左侧固定宽度 200px，右侧自适应。
用 Flexbox 如何实现？请给我代码和解释。
```

3. **CSS 样式问题**
```
我想让一个 div 水平垂直居中，有哪些方法？
请给我 3 种方案并说明各自的优缺点。
```

4. **响应式设计**
```
如何让网页在手机和电脑上都好看？
请给我一个响应式导航栏的示例。
```

**示例对话流程**：

```
你: 我想创建一个个人简历页面，包含照片、姓名、简介和联系方式。
   应该使用哪些 HTML 标签？

AI: [给出结构建议]

你: 很好。现在我想让照片显示为圆形，直径 150px，居中显示。
   CSS 应该怎么写？

AI: [给出 CSS 代码]

你: 为什么要用 border-radius: 50% 而不是具体的像素值？

AI: [解释原理]
```

### 第二阶段：JavaScript + React

**适合问 AI 的问题**：

1. **JavaScript 基础**
```
请解释 JavaScript 中的 let、const 和 var 的区别，
并给我使用场景示例。
```

2. **DOM 操作**
```
如何用 JavaScript 实现点击按钮切换暗黑模式？
请给我完整代码并解释每一步。
```

3. **React 概念**
```
请用简单的比喻解释 React 的组件、Props 和 State。
然后给我一个简单的示例。
```

4. **代码转换**
```
这是我的 HTML/CSS 代码：
[粘贴代码]

请帮我转换成 React 组件，并解释改动的原因。
```

5. **功能实现**
```
我想实现一个 React 组件：点击按钮在亮色和暗色主题之间切换，
并将用户的选择保存到 localStorage。
请分步骤实现并解释。
```

**示例对话流程**：

```
你: 我有一个静态的个人简历 HTML，我想把它改成 React 组件。
   应该如何开始？

AI: [给出改造步骤]

你: 我的简历有个人信息、技能列表、项目经历三个部分。
   应该如何拆分组件？

AI: [给出组件拆分建议]

你: 很好。现在请帮我实现 PersonalInfo 组件，
   接收 name, title, bio 作为 props。

AI: [给出代码]

你: 这段代码中的解构赋值 { name, title, bio } 是什么意思？

AI: [解释原理]
```

### 第三阶段：最终项目

**适合问 AI 的问题**：

1. **架构设计**
```
我要做一个包含首页、关于、项目、联系四个页面的个人主页。
应该如何组织文件结构？使用什么路由方案？
```

2. **性能优化**
```
我的 React 项目首屏加载很慢，如何优化？
请给我具体的优化建议和实现方法。
```

3. **部署问题**
```
我用 Vite 创建的 React 项目，如何部署到 GitHub Pages？
路由在部署后 404 怎么解决？
```

4. **Code Review**
```
这是我的项目代码：
[粘贴关键代码]

请帮我 review 代码，指出：
1. 潜在的 bug
2. 性能问题
3. 可读性改进
4. 最佳实践建议
```

## 🎓 AI 学习路径建议

### 第 1-2 周：谨慎使用

- 主要用 AI 解释概念和语法
- 尝试自己写代码，然后让 AI review
- 理解 AI 建议的原因

### 第 3-4 周：高效协作

- 用 AI 快速生成基础代码框架
- 让 AI 帮助优化和重构
- 用 AI 排查复杂问题

### 整个过程：持续学习

- 遇到不懂的就问 AI
- 但不要直接要答案，要理解原理
- 养成记录 AI 协作的习惯

## ⚠️ AI 使用的边界

### 可以让 AI 帮你做的：

✅ 解释技术概念和代码原理
✅ 生成基础的代码框架
✅ 优化和重构代码
✅ 解决 bug 和错误
✅ 提供设计建议和最佳实践
✅ Review 代码指出问题

### 不应该让 AI 做的：

❌ 一次性生成整个项目
❌ 替你思考技术选型
❌ 完全依赖 AI 而不自己学习
❌ 提交未经测试的 AI 代码
❌ 用 AI 生成的代码而无法解释

## 🔍 验收时的 AI 问答准备

Advisors 可能会问你这些问题：

1. **"这段代码是什么意思？"**
   - 你必须能清楚解释每一行的作用

2. **"为什么用这个方案而不是其他方案？"**
   - 你需要理解技术选型的原因

3. **"如果我要修改这个功能，应该改哪里？"**
   - 你需要理解代码结构

4. **"这段代码是你自己写的还是 AI 生成的？"**
   - 诚实回答，但要证明你理解了

5. **"如果没有 AI，你能实现这个功能吗？"**
   - 展示你的学习成果，而不是依赖
